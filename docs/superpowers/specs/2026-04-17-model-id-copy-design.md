# Copy Model ID on model card — дизайн

## Контекст

Сейчас на `/models` slug модели показывается внизу карточки как маленький `code`-текст.

Проблема:
- пользователю неочевидно, что именно этот slug нужен для агента / OpenAI-compatible клиента;
- визуально строка выглядит как техническая подпись, а не как действие;
- slug отображается с `truncate`, поэтому копировать его глазами/мышкой небезопасно.

При этом docs уже говорят, что для клиента нужно брать именно slug/model id с `/models`.

## Цель

Сделать на карточке модели явное действие, позволяющее сразу скопировать правильный model id без ошибок.

## Принятое решение

Используем явную кнопку **`Copy ID`** рядом со slug внизу карточки модели.

Поведение:
- кнопка копирует **полный `slug` из данных карточки**;
- после успешного копирования кнопка временно меняется на **`Copied`**;
- через короткое время состояние возвращается обратно к **`Copy ID`**.

## Что меняется в UI

На нижней строке карточки:
- slug остаётся видимым как и раньше;
- справа добавляется кнопка `Copy ID`.

Важно:
- кнопка должна быть явно читаемой и понятной;
- не использовать одну только иконку без текста;
- копирование должно брать значение из `slug` prop, а не из DOM-текста, чтобы не скопировать обрезанное значение.

## Scope

В scope входит только:
- UX-кнопка копирования model id на карточке модели;
- локальное визуальное подтверждение `Copied`.

В scope не входит:
- глобальный toast;
- изменение API;
- переработка карточки модели;
- копирование других полей (`hf_repo`, display name и т.д.).

## Проверка после внедрения

Нужно убедиться, что:
- на карточке есть явная кнопка `Copy ID`;
- в clipboard попадает полный slug;
- `Copied` показывается после клика и потом исчезает;
- даже если slug визуально обрезан, в clipboard уходит полное значение.
